# Vibe Coding — 林怡杰作品集网站

这个技能帮你以"先改先看、快速迭代"的方式维护本仓库的作品集网站。所有页面都在**单个 `index.html`** 里(HTML + 内联 `<style>` + 内联 `<script>`,约 1000 行),没有构建步骤、没有依赖安装,改完直接在浏览器打开即可。

## 核心原则(vibe 的来源)

改任何东西前,先守住这套设计语言,别破坏整体气质:

- **风格**:深色、电影感、克制、留白多、慢节奏。不要用鲜艳饱和色、圆润卡通感、密集排版。
- **配色只用 CSS 变量**(定义在 `:root`,`index.html` 顶部 `<style>` 内)。不要硬编码颜色,一律引用变量:
  - `--bg #08080a` / `--bg2` / `--surface` / `--card` — 由深到浅的背景层次
  - `--gold #c9a96a` / `--gold-dim #9a824f` — 唯一强调色,用于点缀、分隔线、hover、关键数字
  - `--red #7a3b3b` — 极少量点缀
  - `--text #ece7dd` / `--text-dim` / `--muted` / `--faint` — 由亮到暗的文字层次
  - `--border` / `--border2` — 细边框
- **字体**:正文 `PingFang SC` 等中文无衬线;标题/数字用 `.serif` 类(Cormorant Garamond 衬线),营造杂志感。
- **氛围元素**别乱删:全局胶片颗粒(`body::before`)、粒子画布(`#particles`)、`scroll-behavior: smooth`。
- **动效**:淡入、缓动、慢。避免弹跳、闪烁、快速位移。
- **语言**:中文文案(`<html lang="zh-CN">`),标点、语气保持简洁高级,不要口语化营销腔。

## 页面结构地图(`index.html` 内的 section)

按顺序排列,靠 HTML 注释 `<!-- ==== XXX ==== -->` 分隔,方便定位:

| 板块 | 锚点 id | 说明 |
|------|---------|------|
| 导航 | `nav`(fixed) | 顶部固定导航 |
| Hero | — | 首屏大标题 |
| 简介 | `#profile` | 个人介绍 |
| 优势 | `#strengths` | 能力亮点 |
| 作品 | `#works` | 视频作品网格(点击弹出播放) |
| 经历 | `#experience` | 时间线/履历 |
| 联系 | `#contact` | 联系方式 |
| 视频弹窗 | modal | 作品点击后的播放层 |

改某板块时,先 `grep -n` 对应注释或 `id` 跳转到位,只动那一段,别牵连全局样式。

## 视频作品管理

仓库根目录的 `.mp4` 就是作品源文件(如 `产品营销.mp4`、`品牌宣传片.mp4`、`明星VLOG.mp4` 等)。`info1~4.mp4` 是占位/小文件。

- **换/加作品**:在 `#works` 板块的作品网格里增删对应的作品卡片,`<video>`/`data-src` 指向根目录文件名(注意中文文件名与空格,`src` 里保持与实际文件名一致)。
- 新增视频文件后,记得在 works 网格里补一张卡片,否则页面上看不到。
- 缩略图/海报如用 `poster=`,确保对应图片也在仓库里。

## 快速迭代流程

1. **定位**:用 `grep -n` 找到要改的板块注释、`id`、或文案关键词,只读那一段(文件大,别整篇重读)。
2. **小步改**:用 `Edit` 精确替换,一次改一处,保持缩进和写法与周围一致。
3. **预览**:直接在浏览器打开 `index.html`(`file://` 即可,无需服务器)。若要本地起服务:`python3 -m http.server 8000` 然后访问 `http://localhost:8000`。需要截图/自动检查时可用 Playwright(环境已内置 Chromium)。
4. **自查**:改样式后扫一眼是否引用了 CSS 变量、是否破坏响应式(`@media` 查询)、深色对比度是否够。
5. **交付**:改动满意后按仓库要求提交到指定分支 `claude/vibe-installation-coding-ommt17` 并推送。除非用户明确要求,否则不建 PR。

## 常见任务速查

- **调色**:改 `:root` 里的变量值,全站自动生效——不要在具体元素上覆盖颜色。
- **改文案**:直接编辑对应 section 的中文文本,保持语气高级克制。
- **加板块**:复制一个现有 `<section>` 的结构骨架,套用相同的容器/间距类,再填内容;在 `nav` 里补锚点链接。
- **调间距/排版**:优先复用已有的 spacing 写法,别引入新的一次性魔法数字。
- **响应式**:每次改布局都检查文件底部的 `@media` 断点,手机端(窄屏)要能正常显示。

## 不要做的事

- 不要引入打包工具、框架、npm 依赖——这是零构建的单文件站。
- 不要把内联 `<style>`/`<script>` 拆成外部文件,除非用户明确要求。
- 不要硬编码颜色/字体,一律走变量与既有类。
- 不要删胶片颗粒、粒子、平滑滚动等氛围元素而不确认。
- 不要一次性大重构;vibe coding 的重点是小步快跑、随时可预览。
